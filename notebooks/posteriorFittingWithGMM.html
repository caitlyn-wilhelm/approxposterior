Letâs fit a GMM model to approximate the true density. Clearly, there are two Gaussian components, but here weâll explore GMMs with up to 10 components to demonstrate how we can use the BIC (https://en.wikipedia.org/wiki/Bayesian_information_criterion) within fitGMM to select the optimal number of components, given the data.

In [4]:

# Fit!
gmm = gmmUtils.fitGMM(data, maxComp=6, covType="full", useBic=True)

Letâs plot the data and our fit to see how we did.

Code based on https://scikit-learn.org/stable/auto_examples/mixture/plot_gmm_selection.html#sphx-glr-auto-examples-mixture-plot-gmm-selection-py example.

In [5]:

fig, ax = plt.subplots(figsize=(9,8))

# Plot data
ax.scatter(data[:,0], data[:,1], s=20)

# Now plot GMM components
colors = ["C%d" % ii for ii in range(len(gmm.means_))]
for i, (mean, cov, color) in enumerate(zip(gmm.means_, gmm.covariances_, colors)):
    v, w = linalg.eigh(cov)

    # Plot an ellipse to show the Gaussian component
    angle = np.arctan2(w[0][1], w[0][0])
    angle = 180. * angle / np.pi  # convert to degrees
    v = 2. * np.sqrt(2.) * np.sqrt(v)
    ell = mpl.patches.Ellipse(mean, v[0], v[1], 180. + angle, color=color)
    ell.set_clip_box(ax.bbox)
    ell.set_alpha(.5)
    ax.add_artist(ell)
